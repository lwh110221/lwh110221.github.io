大纲提取-通信网基础

通信网理论基础知识点笔记

一、通信网络的基本构成和协议体系的基本内容

  • 通信网络的基本构成：
    • 想象一个通信网络就像一个巨大的交通系统。信源就像是出发地，产生要传输的信息；信宿则是目的地，接收信息。中间的各种传输设备和线路就像是道路、桥梁和交通工具。
    • 通信网络主要由终端设备、传输设备和交换设备组成。终端设备比如手机、电脑等，是用户与通信网络交互的接口。传输设备包括电缆、光纤、无线信道等，负责信息的传输。交换设备就像交通枢纽，负责将信息从一个地方转发到另一个地方。
  • 协议体系的基本内容：
    • 通信网络中的协议就像是交通规则，确保信息能够正确、高效地传输。协议体系包括不同层次的协议，就像交通系统中有不同级别的道路规则。
    • 协议体系通常分为物理层、数据链路层、网络层、传输层和应用层等。物理层负责在物理介质上传输比特流；数据链路层负责将比特流组织成帧，并进行差错控制和流量控制；网络层负责将数据包从源地址传输到目的地址；传输层负责端到端的数据传输；应用层则为各种应用程序提供服务。

二、链路层、网络层和传输层的端到端传输协议内容和基本原理

  • 链路层协议：
    • 链路层就像是在一条道路上行驶的车辆，它的任务是确保车辆（数据帧）能够安全、快速地从一个地方行驶到另一个地方。
    • 链路层协议主要包括以太网协议、PPP 协议等。这些协议负责在两个相邻节点之间传输数据帧，进行差错检测和纠正，以及流量控制。例如，以太网协议通过 CSMA/CD 机制避免数据冲突，确保数据的可靠传输。
  • 网络层协议：
    • 网络层就像是交通地图，它的任务是确定数据包从源地址到目的地址的最佳路径。
    • 网络层协议主要包括 IP 协议、ICMP 协议等。IP 协议负责为每个设备分配一个唯一的 IP 地址，并根据 IP 地址进行数据包的路由选择。ICMP 协议则用于报告网络中的错误和异常情况。例如，当一个数据包无法到达目的地时，ICMP 协议会发送一个“目的不可达”的消息给源设备。
  • 传输层协议：
    • 传输层就像是快递员，它的任务是确保数据能够从发送端正确地传输到接收端。
    • 传输层协议主要包括 TCP 协议和 UDP 协议。TCP 协议是一种可靠的传输协议，它通过三次握手建立连接，进行数据的分段和重组，以及差错控制和流量控制。UDP 协议则是一种不可靠的传输协议，它不进行连接建立和差错控制，但是传输速度快。例如，在文件传输时通常使用 TCP 协议，以确保文件的正确传输；而在视频直播时则可以使用 UDP 协议，以减少延迟。

三、排队系统的延时模型，多址技术、路由算法、流量和拥塞控制的基本原理；通信网的拓扑设计

  • 排队系统的延时模型：
    • 想象一个通信网络就像一个繁忙的餐厅，数据包就像是顾客，而网络中的节点就像是服务员。当顾客（数据包）到达餐厅（网络节点）时，如果服务员（节点处理能力）忙不过来，顾客就需要排队等待。
    • 排队系统的延时模型主要考虑数据包在队列中的等待时间和节点处理数据包的时间。等待时间取决于队列的长度和服务时间，而服务时间则取决于节点的处理能力。例如，如果网络中的流量很大，数据包就会在队列中等待很长时间，导致延时增加。
  • 多址技术：
    • 多址技术就像是在一个会议室里，有很多人同时发言，需要一种方法让每个人都能被听到。
    • 多址技术主要包括时分多址（TDMA）、频分多址（FDMA）和码分多址（CDMA）等。TDMA 是将时间分成若干个时隙，每个用户在不同的时隙内发送数据；FDMA 是将频率分成若干个频段，每个用户在不同的频段内发送数据；CDMA 是每个用户使用不同的码序列进行数据传输，通过码序列的正交性来区分不同的用户。例如，在移动通信中，不同的用户可以通过 TDMA、FDMA 或 CDMA 技术同时进行通信。
  • 路由算法：
    • 路由算法就像是在地图上寻找最佳路径，它的任务是确定数据包从源地址到目的地址的最佳路径。
    • 路由算法主要包括静态路由算法和动态路由算法。静态路由算法是由管理员手动配置的路由，适用于小型网络。动态路由算法则是由路由器通过协议自动学习的路由，适用于大型网络。例如，在互联网中，路由器使用动态路由算法，如 OSPF 协议和 BGP 协议，来确定数据包的最佳路径。
  • 流量和拥塞控制的基本原理：
    • 流量和拥塞控制就像是交通管制，当道路上的车辆太多时，需要采取措施来避免交通堵塞。
    • 流量控制的目的是确保发送端发送的数据不会超过接收端的处理能力。拥塞控制的目的是当网络中的流量过大时，采取措施来避免网络拥塞。流量控制和拥塞控制的方法包括窗口机制、慢启动、拥塞避免等。例如，当接收端的缓冲区已满时，它可以通过发送一个“窗口已满”的消息给发送端，让发送端停止发送数据，从而实现流量控制。
  • 通信网的拓扑设计：
    • 通信网的拓扑设计就像是设计一个城市的交通网络，需要考虑到各个节点之间的连接方式和流量分布。
    • 通信网的拓扑结构主要包括总线型、星型、环型、树型和网状型等。不同的拓扑结构具有不同的特点和适用场景。例如，总线型拓扑结构简单，成本低，但是可靠性差；网状型拓扑结构可靠性高，但是成本高，管理复杂。在设计通信网的拓扑结构时，需要根据实际需求进行选择。

四、通信网概述。通信网基本概念，数字通信网和 ISDN 基本概念，移动通信、光通信的基本知识和发展动态

  • 通信网基本概念：
    • 通信网就像是一个巨大的蜘蛛网，将各个地方的人们连接在一起。它由许多节点和连接这些节点的线路组成，可以实现信息的传输和交换。
    • 通信网的主要功能包括语音通信、数据通信、视频通信等。它可以分为有线通信网和无线通信网，也可以分为公用通信网和专用通信网。例如，我们日常使用的电话网、互联网都是通信网的例子。
  • 数字通信网和 ISDN 基本概念：
    • 数字通信网是指采用数字技术进行信息传输和交换的通信网。它具有抗干扰能力强、传输质量高、便于集成和管理等优点。
    • ISDN（综合业务数字网）是一种能够同时提供语音、数据和视频等多种业务的数字通信网。它通过一个统一的网络接口，为用户提供综合的通信服务。例如，用户可以通过 ISDN 网络同时进行电话通话、上网和观看视频。
  • 移动通信、光通信的基本知识和发展动态：
    • 移动通信是指用户可以在移动中进行通信的技术。它主要包括 GSM、CDMA、WCDMA、LTE 等技术。移动通信的发展趋势是高速化、智能化、融合化。例如，5G 移动通信技术可以实现每秒数十吉比特的传输速度，支持大规模物联网连接和低延迟通信。
    • 光通信是指采用光纤作为传输介质的通信技术。它具有传输容量大、传输距离远、抗干扰能力强等优点。光通信的发展趋势是高速化、智能化、集成化。例如，随着光通信技术的不断发展，单根光纤的传输容量已经可以达到数十太比特每秒，而且光通信设备也越来越小型化、智能化。

五、网络可靠性和网络安全。网络可靠性定义、加密传输体系

  • 网络可靠性：
    • 网络可靠性就像是一座坚固的桥梁，能够在各种情况下保证信息的传输。它是指通信网络在规定的条件下和规定的时间内，完成规定功能的能力。
    • 网络可靠性主要通过冗余设计、备份机制、故障检测和恢复等方法来实现。例如，在通信网络中，可以采用双链路备份、设备冗余等方式来提高网络的可靠性。当一条链路或设备出现故障时，另一条链路或设备可以自动接管，保证信息的传输不受影响。
  • 网络安全：
    • 网络安全就像是一个坚固的城堡，保护着网络中的信息不被敌人攻击。它是指保护通信网络中的信息和资源不被未经授权的访问、使用、泄露、破坏或修改。
    • 加密传输体系是网络安全的重要组成部分。它通过对信息进行加密，使得只有拥有正确密钥的用户才能解密并读取信息。加密传输体系主要包括对称加密和非对称加密两种方式。对称加密是指发送方和接收方使用相同的密钥进行加密和解密；非对称加密是指发送方和接收方使用不同的密钥进行加密和解密，其中一个密钥是公开的，另一个密钥是私有的。例如，在网上银行中，用户的信息就是通过加密传输体系进行保护的，确保用户的资金安全。
